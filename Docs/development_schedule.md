- **총 개발 기간:** 12주

## 리스크 및 주의사항

### 고위험 태스크

1. **벌목꾼 군중 시스템 (Week 4-5)**
   - 다수 캐릭터 동시 처리로 성능 이슈 발생 가능
   - 최적화 실패 시 게임 전체 성능에 영향
   - 대응: 조기에 프로파일링, 오브젝트 풀링 우선 적용

2. **통합 시스템 (Task 5.1)**
   - 시각적 통합과 내부 로직 분리 복잡
   - 대응: 명확한 데이터 구조 설계 선행

### 크리티컬 패스

```
1.5 → 2.5 → 4.1 → 4.2 → 5.1
(나무 클릭) → (벌목꾼 기본) → (벌목꾼 AI) → (5종 구현) → (통합 시스템)
```

이 경로의 지연은 전체 일정에 직접 영향을 미침.

### 병렬 작업 가능 영역

| 병렬 가능 태스크 | 설명 |
|------------------|------|
| UI 작업 (3.4, 8.1) | 게임 로직과 독립적으로 진행 가능 |
| 사운드 적용 (8.4-8.6) | 아트 작업과 병렬 진행 가능 |
| 크리티컬 시스템 (5.3-5.4) | 벌목꾼 시스템과 독립적 |
